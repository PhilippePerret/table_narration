memorize_current

(
  • options
)
Number

Defined in /Users/philippeperret/Sites/table_narration/js/ui/>ui_input.js:420

Mémorise le champ courant, s'il est défini, et notamment sa sélection courante, pour pouvoir retrouver le même état. Utilisé lorsque l'on doit par exemple “blurer” un champ de saisie pour choisir un film ou un mot, pour l'insérer ensuite dans ce champ.

Notes

  • La méthode peut être appelée sans qu'il y ait de champ courant. Par exemple, quand on affiche l'aperçu d'un mot (en passant sa souris sur son nom) et qu'on clique sur la définition pour mettre le mot en édition.
  • C'est ensuite la méthode retreive_current qui remet le champ mémorisé dans son état précédent.

Parameters:

  • options Object

    Liste des options à prendre en compte. Pour le moment, ces options se résument à blur qui, si true, blur le champ courant.

Returns:

Number:

id Identifiant (dans targets) du champ mémorisé.

traite_as_acteurs_from_imdb

(
  • value
)
String

Defined in /Users/philippeperret/Sites/table_narration/js/ui/>ui_input.js:368

Traite la donnée +acteurs+ avant sa vérification par la méthode {check_values} dans le cas d'un copié/collé d'une donnée prise dans IMDb.

NOTES

  • On reconnait une donnée IMDb pour un acteur car elle sépare le patronyme de l'acteur du patronyme du personnage par un "...\n". Noter que le patronyme de l'acteur est répété au début, donc la donnée a le format :

    <patronyme acteur> <patronyme acteur> ...
<patronyme personnage>

    Elle sera transformée en :

    <prenom acteur>, <nom acteur>, <prenom perso>, <nom perso>, null

    note: Le null ci-dessus concerne la fonction du personnage dans le film.

Parameters:

  • value String

    La valeur complète contenu dans le champ acteurs.

Returns:

String:

La valeur corrigée, si nécessaire.
